Form and Input Tags

Used to collect data from a user and send it to a server. Form is designated by < form > ... < form > and inside contains the tags to hold the inputs and labels. But can also contain text.
The input tag ( < form ...attributes... />) is a single tag and can contain attributes to hold the value and specifications for that field.
The label tag ( < label > ... < /label >) contains a text label used to specify the field for the input. Looks exactly like text, you can add a for="id" label which must match up exactly with id attribute in the input tag.
The select tag ( < select > ... < /select >) is used for creating a drop down. Drop down values can be created with the option tag ( < option > ... < /option >).
Checkboxes and radio fields can be checked with the checked attribute. No =".." required.

This is a tool to embed content from other websites. It requires the src attribute to link to another url (src="url") and can take other optional attributes like width to adjust the size.
You may also need api keys to access other sites like google maps.

Image Tag < image > ... < /image >

Single tag that allows you to embed an image to the webpage through a url. The src attribute specifies the source of the image.
A bunch of other attribute Tags, but one used commonly is the alt attribute which allows you to set text in case the image fails to load

Aduio/Video Tag < audio/video > ... < /audio/video >

Used to embed audio/video content to a webpage. It has attributes like src, control, etc. Can also provide audio descriptions.
Without the controls attribute you will not be able to see the audo label and is essentially the same as marking it as hidden. However, you can display videos without the controls tag but it won't play unless you have the autoplay attr or the controls attr.
The video tag can show audio or mp4 and audio only shows ogg. Once specifying the aduio or video tag, you can add the single tag source (< source > ... < /source >) to link to the source and descript the content type.

Table Tags

Table tags help you create tables. You can add structure with the optional < thead > ... < /thead >, < tbody > ... < /tbody >, and < tfoot > ... < /tfoot > tags.
These tags help you to structure data for the header columns, the data in the body, and the data in the footer accordingly.
To actually create the table, you first create it with the < table > ... < /table > tag.
Then create a row with the < tr > ... < /tr > tag. Inside the tr tag, you can add individual column data with the < td > ... < /td > tag. Header data should be created with the < th > ... < /th > tag, but can still be in a tr row.
Just using this will not display the bars for the columns and heights. Border should be controlled by css.

Semantic vs non-semantic

Semantic Tags
Semantic tags are used to provide meaning and context to the content they surround. They can help search engines and assitive tech to understand structure and purpose of content.
Some common semantic tags:
  • header
  • nav
  • article
  • section
  • table
  • heading
  • etc
Non-Semantic Tags
Don't provide specific meaning or context but are mostly just used for organizing and formatting content.
E.g. Div tag can contain anything, image, text, form, etc, so it's existence doesn't give us any meaning but can add structure. Some common non-semantic tags:
  • div
  • span
  • etc
